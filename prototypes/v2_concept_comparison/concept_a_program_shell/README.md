# Concept A - Desktop Program Shell

Open `index.html` directly.

This concept tests a conventional desktop-program structure:

- top-level module navigation
- contextual left submenu
- central active workflow panels
- right rail for preview, warnings, and output status

The two active workflow modules are:

- Laser Test Pattern Generator
- Parametric Design / Nameplate Generator

Journal, helper tools, machine context, and settings are visible as supporting surfaces only. This is static HTML/CSS/JS and never writes files or generates machine-control output.
