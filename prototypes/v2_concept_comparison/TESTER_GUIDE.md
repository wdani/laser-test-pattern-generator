# Tester-Guide: v2 Konzeptvergleich

## Ziel

Dieser Prototyp vergleicht zwei mögliche UI-Richtungen für eine spätere v2 Oberfläche des Laser Test Pattern Generators und eines möglichen CNC/Laser Companions:

- **Concept A: Desktop Program Shell**  
  Klassischer Programmaufbau mit Modulen, linker Navigation, zentralem Workflow und rechter Vorschau-/Statusleiste.
- **Concept B: Workshop Companion Command Center**  
  Werkstatt- und joborientierter Aufbau mit aktiven Jobs, Kontext-Helfern, Material-, Maschinen- und Sicherheitsinformationen.

Der Test soll nicht entscheiden, welche Optik am schönsten ist, sondern welche Struktur für reale CNC-/Laser-Workflows klarer, verständlicher und später besser erweiterbar wirkt.

## Wichtig

- Alles läuft lokal im Browser.
- Es wird nichts hochgeladen.
- Feedback wird nicht automatisch gespeichert.
- Nur die Spracheinstellung kann lokal im Browser gespeichert werden.
- Die erzeugte Feedback-Datei muss manuell geteilt werden.
- Der Prototyp erzeugt keine echten Maschinendateien.
- Der Prototyp steuert keine Maschine.
- Der Prototyp ist keine fertige App und kein CAM-Ersatz.

## Testablauf

1. `index.html` öffnen.
2. Intro ansehen oder überspringen.
3. Sprache wählen: **DE** oder **EN**.
4. **Concept A** öffnen.
5. Den **Laser Test Pattern Generator** finden.
6. Den **Parametric Design / Nameplate Generator** finden.
7. Verständlichkeit, Navigation, Vorschau, Status und Helfer prüfen.
8. Zurück zur Übersicht gehen.
9. **Concept B** öffnen.
10. Dieselben Punkte erneut prüfen.
11. `feedback.html` öffnen.
12. Bewertungen und Notizen eintragen.
13. **Feedback erzeugen**.
14. Feedback kopieren oder herunterladen.
15. Feedback manuell per Discord, GitHub, Mail oder Chat teilen.

## Worauf achten?

Bitte besonders auf diese Fragen achten:

- Welches Konzept ist schneller verständlich?
- Wo findet man den Laser Test Pattern Generator leichter?
- Wo findet man Parametric Design / Nameplate leichter?
- Wo sind aktive Workflows und Helfer klarer getrennt?
- Wo wirken Vorschau, Status und Sicherheitswarnungen klarer?
- Welches Konzept skaliert besser für viele spätere Werkzeuge?
- Welches Konzept wirkt besser für nicht-technische Nutzer?
- Welches Konzept fühlt sich mehr wie ein Workshop Companion an?
- Welche Ideen aus beiden Konzepten sollten kombiniert werden?

## Nicht bewerten als

Bitte den Prototyp **nicht** bewerten als:

- fertige App,
- echtes CAM-System,
- echte Maschinensteuerung,
- echten Dateigenerator,
- finale Produktentscheidung.

Der Fokus liegt auf Struktur, Workflow-Klarheit, Bediengefühl, Sprache, Orientierung und Erweiterbarkeit.

## Feedback teilen

Das Feedbackformular erzeugt eine lesbare Markdown-Zusammenfassung. Tester können diese Zusammenfassung kopieren oder herunterladen.

Die heruntergeladene Datei wird nur lokal im Browser erzeugt. Sie wird nicht automatisch versendet. Bitte die Datei oder den kopierten Text anschließend manuell teilen.

# Tester Guide: v2 Concept Comparison

## Goal

This prototype compares two possible UI directions for a future v2 interface of the Laser Test Pattern Generator and a possible CNC/Laser Companion:

- **Concept A: Desktop Program Shell**  
  A classic desktop-style application with modules, left navigation, a central workflow area, and a right preview/status rail.
- **Concept B: Workshop Companion Command Center**  
  A workshop- and job-centered interface with active jobs, contextual helpers, material, machine, and safety information.

The test is not mainly about which visual style looks nicest. The goal is to understand which structure feels clearer, easier to use, and more scalable for real CNC/laser workflows.

## Important

- Everything runs locally in the browser.
- Nothing is uploaded.
- Feedback is not saved automatically.
- Only the language selection may be stored locally in the browser.
- The generated feedback file must be shared manually.
- The prototype does not create real machine files.
- The prototype does not control a machine.
- The prototype is not a finished app and not a CAM replacement.

## Test Flow

1. Open `index.html`.
2. Watch or skip the intro.
3. Select a language: **DE** or **EN**.
4. Open **Concept A**.
5. Find the **Laser Test Pattern Generator**.
6. Find the **Parametric Design / Nameplate Generator**.
7. Check clarity, navigation, preview, status, and helper areas.
8. Go back to the overview.
9. Open **Concept B**.
10. Repeat the same checks.
11. Open `feedback.html`.
12. Enter ratings and notes.
13. Generate feedback.
14. Copy or download the feedback.
15. Share the feedback manually via Discord, GitHub, email, or chat.

## What to Watch For

Please pay special attention to these questions:

- Which concept is easier to understand quickly?
- Where is the Laser Test Pattern Generator easier to find?
- Where is Parametric Design / Nameplate easier to find?
- Where are active workflows and helper tools separated more clearly?
- Where are preview, status, and safety warnings clearer?
- Which concept scales better for many future tools?
- Which concept feels better for non-technical users?
- Which concept feels more like a Workshop Companion?
- Which ideas from both concepts should be combined?

## Do Not Judge It As

Please do **not** judge this prototype as:

- a finished app,
- a real CAM system,
- real machine control,
- a real file generator,
- a final product decision.

The focus is structure, workflow clarity, usability, language, orientation, and future expandability.

## Sharing Feedback

The feedback form generates a readable Markdown summary. Testers can copy or download this summary.

The downloaded file is created only locally in the browser. It is not sent automatically. Please share the file or copied text manually afterwards.
