# Concept Comparison Notes

Use this note while comparing the two static v2 concepts.

## Concept A: Desktop Program Shell

Concept A tests whether the future app should feel like a clean desktop program with top-level modules. The active module owns the left submenu, the center panel, and the right preview/status rail.

This concept is useful for judging:

- whether module navigation is easy to understand
- whether multiple active workflows fit into one program shell
- whether the contextual submenu helps or feels too formal
- whether the right preview/status rail stays readable

## Concept B: Workshop Companion Command Center

Concept B tests whether the future app should feel like a workshop job board. The user chooses a current job, then helpers, output notes, safety context, and material memory adapt around that job.

This concept is useful for judging:

- whether job cards make the current task clearer
- whether helpers feel naturally connected to a selected job
- whether the dashboard feels more like a CNC/Laser Companion
- whether the interface feels helpful or too busy

## Active Workflows Included

- Laser Test Pattern Generator
- Parametric Design / Nameplate Generator

The Parametric Design workflow is represented as a simple future generator for CNC/laser-ready design previews from parameters such as shape, size, text, motif, operation intent, and output target. It does not generate real machine files.

## Suggested Comparison Questions

- Which concept is easier to understand at first glance?
- Which concept feels better for multiple tools?
- Which concept makes the active job clearer?
- Which concept handles helper tools better?
- Which concept feels more like a future CNC/Laser Companion?
- Which concept feels less overwhelming?
- Which concept makes safety and output verification more visible?

## Known Limitations

- Static HTML/CSS/JS only.
- No Python backend calls.
- No real file generation.
- No machine-control output.
- No persistent settings.
- No v1.x generator behavior changes.
