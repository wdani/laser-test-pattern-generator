v2 Concept Comparison

This is a static design study for comparing two possible v2 UI directions for the Laser Test Pattern Generator and a future CNC/Laser Companion.

Concept A

Desktop Program Shell

A classic desktop app with modules, a left submenu, a central workflow area, and a right preview/status rail.

Concept B

Workshop Companion Command Center

A job/workshop centered assistant with active jobs, helper tools, and material, machine, and safety context around the current task.

Passive/helper tools

Helpers support the active job.

Tools such as the V-bit Calculator, Safety Checklist, Material Journal, Cost Calculator, and SVG Detail Checker should help the current workflow. They are not the main workflow being tested.

Tester checklist

Suggested test order

  1. Open Concept A.
  2. Find Laser Test Pattern.
  3. Find Parametric Design / Nameplate.
  4. Check whether the preview, status, and helper areas are understandable.
  5. Open Concept B.
  6. Repeat the same checks.
  7. Compare which concept feels clearer, less overwhelming, and more useful as a future CNC/Laser Companion.
Comparison questions

Questions for reviewers

First impression
  • Which concept is easier to understand at first glance?
  • Which concept feels less overwhelming?
  • Which concept would be better for non-technical users?
Workflow fit
  • Which concept makes it easier to find the Laser Test Pattern Generator?
  • Which concept makes it easier to find the Parametric Design / Nameplate workflow?
  • Which concept feels better for many future tools?
  • Which concept feels more like a workshop companion?
