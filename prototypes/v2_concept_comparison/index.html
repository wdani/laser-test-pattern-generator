What is being compared?

Two UI directions for a future v2 interface.

Both concepts show the same core ideas with different operating models: one as a desktop program with modules, one as a workshop companion with current jobs and contextual helpers.

Active workflows

These two workflows should be tested in both concepts.

Laser Test Pattern Generator Parametric Design / Nameplate Generator
Helpers / passive tools

Helpers support the active job.

V-bit Calculator, Safety Checklist, Material Journal, Cost Calculator, and SVG Detail Checker are supporting surfaces. They should help without replacing the main workflow.

Recommended test flow

Test both concepts in roughly the same way.

  1. Open Concept A.
  2. Find the Laser Test Pattern Generator.
  3. Find the Parametric Design / Nameplate Generator.
  4. Check whether preview, status, and helpers are understandable.
  5. Open Concept B.
  6. Repeat the same checks.
  7. Use the feedback form to compare which concept feels clearer, less overloaded, and more useful as a future CNC/Laser Companion.
Comparison questions

What testers should watch for

First impression
  • Which concept is easier to understand at first glance?
  • Which concept feels less overwhelming?
  • Which concept would be better for non-technical users?
Workflow fit
  • Where is the Laser Test Pattern Generator easier to find?
  • Where is Parametric Design / Nameplate easier to find?
  • Which concept handles many future tools better?
  • Which concept feels more like a Workshop Companion?
Prototype limits

Static prototype, no machine files.

  • Static HTML/CSS/JS.
  • No backend.
  • No network connection and no upload.
  • No real file generation.
  • No machine files and no machine output.
  • No change to v1.x generator behavior.
  • No Tauri implementation yet.
