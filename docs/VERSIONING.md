# Versioning

The project uses a semantic-versioning style: `MAJOR.MINOR.PATCH`.

- `v1.3.0` is a feature/minor release.
- `v1.3.1` is a small bugfix, docs update, or small polish release.
- `v1.4.0` is the next real feature release.
- `v2.0.0` is for a larger refactor or breaking structural change.

Optional pre-release versions can be used for experiments before a stable
release:

- `v1.4.0-alpha.1`
- `v1.4.0-beta.1`
- `v1.4.0-rc.1`

The visible app version may stay at the current release while documentation-only
or planning branches are being prepared.
